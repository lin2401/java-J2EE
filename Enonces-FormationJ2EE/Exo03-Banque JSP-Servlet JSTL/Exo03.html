Java JEE - Exercice 3 - MVC

Association des deux exercices afin de réaliser du MVC. On fera aussi usage de la JSTL.

MVC = Le Modèle (vos beans Compte, Utilisateur), Les vues (vos JSPs), Les contrôleurs (vos servlets).

D'une vue (JSP), j'ai le droit d'aller vers un contrôleur (Servlet) en amenant des modèles (vos beans/entités/données), d'un contrôleur j'ai le droit d'aller vers une vue (JSP) en amenant des modèles (vos beans/entités/données).

Important : cet exercice n'est pas à reproduire dans un projet réel, il est en l'état absolument pas sécurisé.

1 Installation

Importation du projet

Nous allons réunir les deux exercices précédents en un seul afin de répartir les rôles entre JSP et Servlet.

Vous pouvez

  • continuez dans le même exercice, dans ce cas ajoutez la dépendance suivante (qui nous servira pour la JSTL) à votre fichier pom.xml 
      ...
  <dependency>
    <groupId>javax.servlet</groupId>
    <artifactId>jstl</artifactId>
    <version>1.2</version>
  </dependency>
  ...
  • ou importez celui qui est dans le dossier de l'exercice (en03.mvc.jstl).

Si vous modifiez le fichier pom.xml, n'oubliez pas de faire un clic droit sur votre projet puis Maven - Update Project... afin qu'Eclipse prenne en compte les modifications.

2 Client

Servlet Client

Editez la servlet client, modifiez son code afin qu'elle réaliser les opérations suivantes :

  • Se connecter à la base de données
  • Exécuter une requête qui récupérer la liste de tous les clients.
  • Place cette liste en request.
  • Se déconnecter de la base de données
  • Rediriger vers la JSP client sans perdre vos données (voir suite de l'exercice) 
      ...
  RequestDispatcher dispatcher = request.getRequestDispatcher("NomEtCheminDeLaPageOuVousVoulezAller.jsp");
  dispatcher.forward(request, response);
  ...

JSP Client

Editez la JSP client, modifiez son code afin qu'elle réaliser les opérations suivantes :

  • Récupèrer de la request la liste des clients
  • Afficher cette liste en faisant usage des JSTL
  • Quand on va cliquer sur un client on veut aller vers la servlet compte (MVC oblige).

Test

Si vous avez importez le projet de l'énoncé, n'oubliez pas d'ajouter votre projet au serveur en sélectionnant le serveur puis en cliquant sur le bouton Add

Lancez votre serveur, puis allez sur l'URL qui caractérise votre contrôleur : http://localhost:8080/NomDeVotreContext/NomDuControleur

Rappels :

  • ATTENTION aux majuscules/minuscules
  • le NomDeVotreContext est spécifié dans les propriétés du projet (si vous avez importez le projet ce sera en03.mvc.jstl). Pour retrouver son nom de context root, faire un clic droit sur son projet puis Properties, allez dans Web Project Settings.
  • le NomDuControleur est l'url mapping de la servlet (si vous avez importez c'est ServletClient).

3 Compte

Servlet Compte

Editez la servlet compte, modifiez son code afin qu'elle réaliser les opérations suivantes :

  • Se connecter à la base de données
  • Exécuter une requête qui récupérer la liste des comptes du client dont l'id est dans la request en tant qu'attribut.
  • Place cette liste en request.
  • Se déconnecter de la base de données
  • Rediriger vers la JSP compte sans perdre vos données (voir suite de l'exercice) 
      ...
  RequestDispatcher dispatcher = request.getRequestDispatcher("NomEtCheminDeLaPageOuVousVoulezAller.jsp");
  dispatcher.forward(request, response);
  ...

JSP Compte

Reprenez votre JSP compte (ou éditez la si vous avez fait l'import), modifiez son code afin qu'elle réaliser les opérations suivantes :

  • Récupèrer de la request la liste des comptes
  • Afficher cette liste en faisant usage des JSTL

Attention : vous êtes en polymorphisme sur la manipulation de la liste des comptes. N'oubliez pas qu'il existe plusieurs types de compte.

Deux solutions :

  • Utiliser la JSTL pour faire des if et des tests sur le nom des classes de vos objets Compte
  • Créer un objet CompteGenerique qui sera fabriqué par votre Servlet Compte afin de simplifier l'affichage.

Test

Lancez votre serveur, puis allez sur l'URL qui caractérise votre contrôleur client, clicquez sur un des boutons qui vous amènera sur le contrôleur compte.

Regardez vos URLs, quand on respecte le MVC, on ne doit jamais voir d'URL de JSP (car on passe toujours par des contrôleurs).
